■ 요청내용
1. 위치 마스터 정보
- 각 Zone에 대한 소속 점,층, 이름이 존재하는 마스터 샘플 데이터
2. 인터페이스를 위한 정보
- 마스터 정보가 있는 테이블명, 서버정보,계정정보 등

위 요청사항에 대한 답변 드립니다.

먼저 저희 시스템의 서버 구조를 간단하게 설명드리자면
신세계 LBS 시스템은
1. 8개의 Rest(JSON) API 서버
- AAMS : 권한 관리 서버
- BMS : 비콘 관리 서버
- CBMS : 회사, 브랜드, 브렌치 관리 서버
- DEVMS : 단말 관리 서버
- LMS : 로그 관리 서버
- MMS : 지도 관리 서버
- ZCMS : 존 캠페인 관리 서버
- ZMS : 존 관리 서버
2. 2개의 관리(UI) 서버
- Admin : 관리자 서버
- CPGN : 데모 캠페인 서버
로 이루어져 있습니다.

여기서 캠페인과 관련있는 서버는
- CPGN : 캠페인 CRUD 입력 화면
- ZCMS : 캠페인 CRUD 처리 로직
- ZMS : 캠페인 등록시 존 선택
주로 3개이며, 이중에 CPGN은 소스와 스키마를 모두 제공하니 참고하여 개발하시면 됩니다.
여기서 각 서버들 간에는 내부적으로도 모두 Rest 오픈 API 방식으로 인터페이스하고 있습니다.
(즉, DBtoDB 등의 다른 인터페이스 방식은 제공하지 않고 있습니다.)

<캠페인 등록 2가지 방법>
1. CPGN 처럼 캠페인 등록시 존 선택 팝업을 통해 존을 1개씩 선택하는 방법
2. 존에 카테고리(대/중/소분류)를 입력하고, 캠페인 등록시 카테고리 정보를 전달하여 다수의 캠페인을 매핑하는 방법

<Rest API 사용 방법>
1. 해당 서버에 /v1/aams/token.do(POST)를 호출하여 access_token 을 얻어온다.
2. 해당 서버에 HTTP 헤더에 access_token:"" 과 request_type:"2" 를 보낸다. 

<CPGN 설치방법>
1. Eclipse, MySql, MAVEN, Tomcat 설치
2. MySql에서 CPGN/export_ddl_campaign_db.sql 실행
3. 터미널에서 CPGN/install_jar.sh 또는 install_jar.bat 실행
4. Eclipse에서 MAVEN Update
5. /CPGN/src/main/webapp/WEB-INF/spring/config/context-datasource.xml 에 MySql 계정 입력
6. Tomcat 실행
7. http://localhost:8090/campaign/login.do 접속 (로그인 계정은 모르면 연락 바람)

<첨부 파일>
1. 캠페인 등록시(ZCMS 호출시) 전달 데이타 설명
2. 신세계 LBS Rest API 문서
3. CPGN 설치 파일
