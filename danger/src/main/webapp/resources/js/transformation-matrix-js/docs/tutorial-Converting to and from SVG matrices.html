Converting to and from SVG matrices

A SVGMatrix has compatible properties (a-f) which can either be applied manually:

var svgMatrix = ...obtained from somewhere...,
    matrix;

matrix = new Matrix(context).setTransform(svgMatrix.a, svgMatrix.b, svgMatrix.c, svgMatrix.d, svgMatrix.e, svgMatrix.f);

or by using the new static fromSVGMatrix() method which acts as a short-hand for the same:

var matrix = Matrix.fromSVGMatrix( svgMatrix );

If you want to replicate a transformation sequence applied to a SVG shape, you can use the static method fromSVGAnimList().

Example - If you have this SVG in your markup:

<svg xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">
    <rect x="50" y="50" height="150" width="150"
        transform="translate(5 -50) rotate(10 75 75) skewX(10)">
    </rect>
</svg>

You can obtain the shape's (the rect in this case) transformation list like this:

var shape = document.querySelector("svg > rect"),
    list = shape.transform.baseVal;

Feeding this to the fromSVGAnimList() method produces a matrix that will produce the same transform in canvas 2D context or with your point or point arrays:

var matrix = Matrix.fromSVGAnimList( list );

A matrix can be converted into a SVGMatrix using:

var svgMatrix = matrix.toSVGMatrix();

As SVGMatrix doesn't allow one to set an absolute transform the process is using decomposition of the current matrix, where the decomposed values then gets applied to the SVGMatrix in QR order (translate - rotate - scale - skew).

SVG matrices can also be used directly with canvas in newer browsers.
